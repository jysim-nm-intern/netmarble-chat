## Phase별 핵심 개선 사항

**Phase 1 → Phase 2:**
1. **메시지 저장소 분리**: MySQL → MongoDB, cursor-based 페이징 P99 7ms 달성
2. **서버 역할 분리**: STOMP 트래픽과 REST 조회 트래픽 격리
3. **동시 접속 5배 확대**: 200명 → 1,000명에서도 핵심 지표 모두 통과

**Phase 2 → Phase 3:**
1. **STOMP 브로커 교체**: SimpleBroker(200명/방) → RabbitMQ(500명/방, 22.9× fan-out, 1,593K 메시지 수신)
2. **수평 확장 기반 마련**: Nginx LB → chat-server ×2, 크로스 인스턴스 메시지 전파 동작
3. **이미지 기능 추가**: 로컬 파일 + Nginx 서빙, 업로드 p(95) 74ms 달성
4. **서버 튜닝**: JVM `-Xms512m -Xmx2g`, HikariCP 커넥션풀 50으로 확대

**Phase 3 → Phase 4:**
1. **NestJS 마이그레이션**: SDD 명세 기반으로 Java → TypeScript 전환, 기능적 동등성 100% 확인
2. **WebSocket 성능 개선**: 연결 속도 7.5× 빠름, 세션 안정성 22% 향상 (tail latency 40% 감소)
3. **SDD 기술 독립성 실증**: 동일 k6 테스트로 양쪽 서버 검증 성공
4. **REST API 튜닝**: N+1 쿼리 제거 + 커넥션풀 50 + DB 인덱스 → HTTP avg 34% 개선, Java 대비 REST 격차 5.4×→3.5×로 축소
