# 부하테스트 결과 리포트

**실행일:** 2026-03-05
**도구:** k6 v1.6.1
**서버 환경:** Docker Compose (Phase 3 — 다중 인스턴스 + RabbitMQ)

---

## 테스트 환경

| 항목 | 설정값 |
|------|--------|
| 아키텍처 | **Nginx**(8888, LB) → **chat-server** ×2 (STOMP+REST) + **api-server**(8081, REST 전용) |
| DB | MySQL 8.0 (유저/채팅방) + **MongoDB** (메시지) + **Redis** (읽음 상태) |
| 메시지 브로커 | **RabbitMQ** (STOMP Relay, 크로스 인스턴스 메시지 전파) |
| 이미지 저장소 | 로컬 파일 시스템 (Docker shared volume `/data/uploads`) |
| 서버 실행 방식 | `java -jar app.jar` (Docker 컨테이너) |
| JVM 옵션 | 기본 Spring Boot 설정 |
| 컨테이너 구성 | nginx, mysql, mongodb, redis, rabbitmq, chat-server ×2, api-server, client |

---

## Phase 1 → Phase 2 → Phase 3 아키텍처 비교

| 항목 | Phase 1 | Phase 2 | Phase 3 |
|------|---------|---------|---------|
| 서버 | 단일 server | chat-server + api-server | Nginx LB → chat-server ×2 + api-server |
| 메시지 저장 | MySQL (JPA) | MongoDB (비동기 쓰기) | MongoDB (동일) |
| 메시지 조회 | chat-server 직접 | api-server (MongoDB cursor) | api-server (동일) |
| 읽음 처리 | MySQL | Redis 원자적 카운트 | Redis (동일) |
| 세션 관리 | HttpSession | JWT + Stateless | JWT + Stateless (동일) |
| STOMP 브로커 | SimpleBroker | SimpleBroker | **RabbitMQ STOMP Relay** |
| 이미지 | 미지원 | 미지원 | **로컬 파일 + Nginx 서빙** |
| 수평 확장 | 불가 | 제한적 | **chat-server N개 확장 가능** |

### 핵심 개선 사항

**Phase 1 → Phase 2:**
1. **메시지 저장소 분리**: MySQL → MongoDB, cursor-based 페이징 P99 7ms 달성
2. **서버 역할 분리**: STOMP 트래픽과 REST 조회 트래픽 격리
3. **동시 접속 5배 확대**: 200명 → 1,000명에서도 핵심 지표 모두 통과

**Phase 2 → Phase 3:**
1. **STOMP 브로커 교체**: SimpleBroker(200명/방) → RabbitMQ(500명/방 브로드캐스트 확인)
2. **수평 확장 기반 마련**: Nginx LB → chat-server ×2, 크로스 인스턴스 메시지 전파 동작
3. **이미지 기능 추가**: 로컬 파일 + Nginx 서빙, 업로드 p(95) 74ms 달성
